# PhotoCheckDate

**PhotoCheckDate** — це зручна та сучасна програма з графічним інтерфейсом на базі Tkinter для автоматичного відновлення та упорядкування дат створення медіафайлів (фотографій, відео та аудіозаписів), у яких було втрачено або пошкоджено системні атрибути дати (наприклад, після копіювання, відновлення з резервних копій чи перенесення між пристроями).

Проект пов'язаний із репозиторієм: [https://github.com/OlegSkalGit/PhotoCheckDate](https://github.com/OlegSkalGit/PhotoCheckDate)

---

## 🌟 Основні можливості

1. **Багатоформатність**: підтримка найпоширеніших форматів зображень (JPEG, PNG, TIFF, HEIC/HEIF, DNG, WEBP, GIF), відео (MP4, MOV, 3GP, AVI) та аудіо (MP3).
2. **Робота безпосередньо з файлами (In-place)**: дати змінюються прямо у поточних файлах в обраній папці, без необхідності копіювання чи створення дублікатів.
3. **Гнучкий алгоритм пошуку дат**: програма застосовує пріоритетний пошук правильної дати створення на основі назви файлу або його внутрішніх метаданих.
4. **Коректна робота з Windows OS**: на відміну від стандартного `os.utime`, який змінює лише час зміни, PhotoCheckDate за допомогою низькорівневих системних викликів Win32 API (`ctypes`) коригує як дату зміни (`Modification Time`), так і оригінальну дату створення (`Creation Time`).
5. **Сучасний інтерфейс**:
   - Підтримка світлої та темної тем з плавним перемиканням.
   - Детальний журнал (консоль логування) для відстеження обробки кожного файлу в реальному часі.
   - Вікно детальної довідки, що автоматично завантажується безпосередньо з цього файлу `README.md`.
   - Автоматичне встановлення необхідних залежностей при першому запуску.
6. **Автоматичне збереження налаштувань**: поточна тема інтерфейсу (світла/темна) та шлях до папки автоматично зберігаються у файлі `config.json` та відновлюються при наступному запуску програми.

## 🛠️ Як працює алгоритм визначення дати

Програма намагається визначити точну дату та час створення файлу за наступним пріоритетним списком:

### 1. Ім'я файлу (Найвищий пріоритет)
Найчастіше камери смартфонів або месенджери зберігають дату зйомки у назві файлу. Програма аналізує ім'я файлу за допомогою регулярних виразів у пошуках таких шаблонів:
*   **Дата та час**: `YYYYMMDD_HHMMSS` (наприклад, `IMG_20231024_153022.jpg`, `VID_20250606_120000.mp4`, `PXL_20240101_091530.jpg`).
*   **Розділені дати**: шаблони з дефісами, крапками чи підкресленнями, такі як `YYYY-MM-DD-HHMMSS` або `YYYY.MM.DD_HH.MM.SS`.
*   **Лише дата**: `YYYY-MM-DD` або `YYYYMMDD` (у цьому випадку час автоматично встановлюється на полудень — `12:00:00`, щоб уникнути часових зсувів).
*   **Unix-таймстемп**: 10-, 13- (у мілісекундах), 16- (у мікросекундах) або 19-значні (у наносекундах) числа у назві файлу (наприклад, `1744009451972874`), які перетворюються на відповідну дату й час.
*   **Скорочена дата (6-значна)**: шаблони `YYMMDD_HHMMSS` (наприклад, `060423_153556.jpg` -> `2006-04-23 15:35:56`).
*   **Текстова дата кирилицею**: `DD [Місяць] YYYY` (наприклад, `16 серпня 2006` чи `16 августа 2006` -> `2006-08-16 12:00:00`).

### 2. EXIF метадані (Для фотографій)
Якщо назва файлу не містить дати, програма переходить до аналізу внутрішніх метаданих зображення (EXIF) для файлів форматів JPEG, PNG, TIFF, DNG, WEBP, а також HEIC/HEIF (за допомогою бібліотеки `pillow-heif`):
*   Аналізуються теги `DateTimeOriginal` (оригінальна дата зйомки), `DateTimeDigitized` (дата оцифрування) та загальний тег `DateTime`.

### 3. Метадані MP4/MOV/3GP (Для відео)
Програма виконує бінарний аналіз структури файлів MP4, MOV та 3GP без використання важких сторонніх утиліт (FFmpeg тощо):
*   Вона шукає атом `moov` та його заголовок `mvhd` (Movie Header).
*   Видобуває часову позначку QuickTime-епохи (кількість секунд з 1 січня 1904 року) та конвертує її у локальний час.

### 4. Метадані ID3v2 (Для аудіо MP3)
Аналізуються аудіофайли на наявність тегів версії ID3v2.3 та ID3v2.4:
*   Використовуються теги `TDRC` (Recording Time), `TYER` (рік), `TDAT` (день/місяць) та `TIME` (години/хвилини).

### 5. Системні атрибути файлу (Фінальний варіант)
Якщо назва файлу не містить дати та відсутні внутрішні метадані:
*   Програма зчитує системний час створення (`ctime`) та час зміни (`mtime`) оригінального файлу.
*   Обирається **найстаріша** з цих двох дат (найменший таймстемп), що дозволяє максимально точно відновити первинну дату файлу перед його перенесенням чи копіюванням.

## 🖥️ Опис інтерфейсу користувача

Інтерфейс програми складається з таких елементів:
*   **Панель інструментів (зверху)**:
    - Кнопка **«Довідка 📚»**: Відкриває вікно з цим текстом інструкції.
    - Кнопка **«Темна тема 🌙 / Світла тема ☀️»**: Дозволяє миттєво перемикати кольорову палітру додатка.
*   **Поле «Папка з файлами»**:
    - Вказується шлях до папки, яка містить фотографії, відео та аудіозаписи, дати яких потрібно оновити.
    - Під полем розміщено короткий опис: *«Оберіть папку, в якій буде змінено дату створення та модифікації файлів прямо на місці (in-place).»*
*   **Консоль логування (Журнал)**:
    - Відображає детальну інформацію про кожен оброблений файл: знайдену дату, джерело дати (ім'я файлу, EXIF тощо) та результат зміни.
*   **Кнопка «Запустити обробку 🚀»**:
    - Розпочинає процес сканування папки та оновлення атрибутів файлів на місці.

## ⚙️ Системні вимоги та запуск

*   **Операційна система**: Windows (рекомендовано для повної підтримки коригування дати створення) або macOS/Linux (підтримується коригування дати зміни).
*   **Python**: версія 3.8 або вище.

### Інструкція із запуску:
1. Завантажте або склонуйте проект у робочу директорію.
2. Відкрийте термінал у папці проекту та запустіть команду:
   ```bash
   python main.py
   ```
3. Програма самостійно перевірить наявність бібліотек `Pillow` та `pillow-heif` і, якщо їх немає, автоматично встановить їх перед запуском інтерфейсу користувача.
